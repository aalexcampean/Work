Em uma execução transacional se qualquer teste falhar a execução é abortada e todos os demais testes são marcados como Bloqueados.

Por exemplo, vamos supor que nós temos os testes A, B, C que rodam nesta mesma ordem. Se a opção de Execução Transacional está marcada e o teste A falha então A é marcado com status 'Falhou'. E os testes B e C não são executados mas têm seus status atualizados para 'Bloqueados'.

Esta opção é útil em situações onde você precisa salvar recursos caso qualquer teste falhe ou quando você tem um cenário onde o resultado de uma execução de teste fornece informação para outros testes.

Para mais informações veja a página Wiki do TestLink plug-in Hudson http://wiki.hudson-ci.org/display/HUDSON/TestLink+Plugin.
